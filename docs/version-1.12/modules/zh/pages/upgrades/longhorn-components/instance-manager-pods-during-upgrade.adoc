= 升级期间的实例管理器 Pod
:revdate: 2026-01-08
:page-revdate: {revdate}
:current-version: {page-component-version}

在实时升级后，您可能会注意到一些*旧实例管理器 Pod*仍在运行。这是预期的行为，*不是一个错误*。

== 为什么会发生这种情况

{longhorn-product-name} 在实时升级期间使用以下策略来管理实例管理器 Pod：

* 一旦没有引擎或副本进程在旧实例管理器 Pod 中运行，旧实例管理器 Pod 会立即被清理。
* 在*实时引擎升级*期间：
** 副本进程在使用升级后的引擎镜像的新实例管理器 Pod 中重新创建。
** 为了避免中断 I/O，{longhorn-product-name} 在卷附加时无法在新实例管理器 Pod 中使用升级后的引擎镜像启动新的引擎进程。相反，旧实例管理器 Pod 继续管理新引擎进程的生命周期，直到卷被分离。因此，新引擎进程在旧实例管理器 Pod 中运行。

因此，只要旧实例管理器 Pod 托管一个活动的引擎进程，它就会保持运行。一旦卷被分离且没有引擎或副本进程，{longhorn-product-name} 将自动清理旧实例管理器 Pod。

== 如何检查哪些卷正在使用旧实例管理器 Pod

您可以从{longhorn-product-name} UI 检查哪些卷仍在使用旧实例管理器 Pod：

* 导航到 *卷* > *名称* > *卷详细信息* > *实例管理器*。

== 如何完全清理旧实例管理器 Pod

要允许{longhorn-product-name}清理所有旧实例管理器 Pod：

. 分离所有仍在使用旧实例管理器 Pod 的卷。
. 一旦没有引擎或副本进程，{longhorn-product-name} 将自动去除旧实例管理器 Pod。
