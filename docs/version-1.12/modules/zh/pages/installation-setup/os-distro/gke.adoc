= Google Kubernetes Engine (GKE)
:revdate: 2025-06-27
:page-revdate: {revdate}
:current-version: {page-component-version}

要在使用 Google Kubernetes Engine 配置的集群上操作 {longhorn-product-name}，需要一些额外的配置。

. GKE 集群必须使用 `Ubuntu` 操作系统，而不是 `Container-Optimized` 操作系统，以满足 {longhorn-product-name} 的 `open-iscsi` 依赖关系。
. GKE 要求用户手动声明自己为集群管理员，以启用基于角色的访问控制。在安装 {longhorn-product-name} 之前，请运行以下命令：
+
[subs="+attributes",shell]
----
 kubectl create clusterrolebinding cluster-admin-binding --clusterrole=cluster-admin --user=<name@example.com>
----
+
其中 `name@example.com` 是 GCE 中的用户账户名称。 它区分大小写。有关更多信息，请参见 https://cloud.google.com/kubernetes-engine/docs/how-to/role-based-access-control[此文档]。
