== 从 v1.10.x 升级

{longhorn-product-name} 仅支持从 v1.10.x 升级到 v{current-version}。对于其他版本，请先升级到 v1.10.x。

对于在作为 Rancher 应用安装的 {longhorn-product-name} 进行隔离升级时，您需要修改镜像名称并删除注册表 URL 部分。

对于在作为 Rancher 应用安装的 {longhorn-product-name} 进行隔离升级时，您需要修改镜像名称并删除注册表 URL 部分。例如，镜像 `registry.example.com/longhorn/longhorn-manager:v{patch-version}` 在 Longhorn 镜像部分更改为 `longhorn/longhorn-manager:v{patch-version}`。有关更多信息，请参见隔离安装步骤 xref:installation-setup/installation/airgapped-environment.adoc#_using_a_rancher_app[这里]。

=== 使用 {rancher-product-name} UI 升级

以下过程描述了如何在 {rancher-product-name} 上升级 {longhorn-product-name} APP。

. 转到 *APP > 储存库*。

. 为储存库提供一个名称，例如 `suse-storage`。
. 在 *OCI 储存库主机 URL* 字段中输入：

`oci://dp.apps.rancher.io/charts/suse-storage`

image::screenshots/install/add-repo.png[添加储存库, width=100%]

. 点击 *创建* 按钮。

image::screenshots/install/add-repo-success.png[储存库已添加, width=100%]

+
image::screenshots/install/find-chart.png[查找图表, width=100%]

. 点击图表，然后点击 *安装*。
+
image::screenshots/install/install-chart.png[安装图表, width=100%]

. 在下一页，将 `global.imagePullSecrets` 设置为 `application-collection`，然后点击 *安装*。
+
请参考 https://docs.apps.rancher.io/get-started/authentication/#kubernetes[身份验证文档] 进行凭证设置。
+

. 这将升级 `suse-storage` `{current-version}`
+
image::screenshots/install/update-repo.png[更新储存库, width=100%]

=== 使用 Helm 升级

升级 {longhorn-product-name} Helm 图表涉及将您的部署更新到较新版本或在 Longhorn 社区镜像和 {longhorn-product-name} 镜像之间切换。

升级的一般 Helm 命令是：

[subs="+attributes",shell]
----
helm upgrade longhorn oci://dp.apps.rancher.io/charts/suse-storage \
  --namespace longhorn-system \
  --version <version> \ # Replace with the version you would like to upgrade to
  --set global.imagePullSecrets=<PULL_IMAGE_SECRET> \
  -f values.yaml
----

[NOTE]
====
要创建一个密钥，请遵循 https://docs.apps.rancher.io/get-started/authentication/#kubernetes[AppCo 文档]。
====

或者，如果在没有特定 `values.yaml` 文件的情况下升级并仅更改版本：

[subs="+attributes",shell]
----
helm upgrade longhorn oci://dp.apps.rancher.io/charts/suse-storage \
  --namespace longhorn-system \
  --version <version> \ # Replace with the version you would like to upgrade to
  --set global.imagePullSecrets=<PULL_IMAGE_SECRET>
----

* *升级路径*：请参考官方 xref:upgrades/upgrades.adoc[{longhorn-product-name} 升级文档] 以获取特定版本到版本的升级路径、前提条件和升级后验证步骤。这对于确保顺利升级和数据完整性非常重要。
* *配置更改*:如果您有自定义配置，请确保将其迁移到新图表版本，并与任何默认更改合并。

=== 使用 Fleet 升级

在您的 GitOps 储存库的 `helm.version` YAML 文件中更新 `fleet` 的值。

[subs="+attributes",yaml]
----
helm:
  repo: https://charts.longhorn.io
  chart: longhorn
  version: v{patch-version} # Replace with the {longhorn-product-name} version you would like to upgrade to
  releaseName: longhorn
----

=== 使用 Flux 升级

在您的 GitOps 储存库的 `spec.chart.spec.version` YAML 文件中更新 `HelmRelease` 的值。

[subs="+attributes",yaml]
----
spec:
  chart:
    spec:
      chart: longhorn
      reconcileStrategy: ChartVersion
      sourceRef:
        kind: HelmRepository
        name: longhorn
      version: v{patch-version} # Replace with the {longhorn-product-name} version you would like to upgrade to
----

=== 使用 Argo CD 升级

在您的 GitOps 储存库的 `targetRevision` YAML 文件中更新 `Application` 的值。

[subs="+attributes",yaml]
----
spec:
  project: default
  sources:
    - chart: longhorn
      repoURL: https://charts.longhorn.io
      targetRevision: v{patch-version} # Replace with the {longhorn-product-name} version you would like to upgrade to
----
