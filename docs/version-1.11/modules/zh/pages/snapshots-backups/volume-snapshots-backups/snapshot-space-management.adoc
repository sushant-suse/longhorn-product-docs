= 快照空间管理
:revdate: 2025-06-27
:page-revdate: {revdate}
:current-version: {page-component-version}

Longhorn 允许您为每个卷配置最大快照数量和最大快照总大小。这两个设置不考虑已删除的快照、备份镜像和卷头快照。当达到这些限制时，您必须删除快照才能创建新的快照。

在早期版本中，最大快照数量不可配置（值为 250），并且没有办法限制快照空间使用。

== 设置

=== 全局设置

*快照最大数量*:您可以为每个卷创建的最大快照数量。

您必须指定一个介于“2”和“250”之间的值。Longhorn 需要至少两个快照才能正常工作，特别是在卷重建期间。在合并现有快照时会创建一个快照，而在重建过程中会创建另一个快照。
默认值为“250”。

当您创建一个卷而不更改 `.Spec.SnapshotMaxCount` 的默认值时，Longhorn 会应用 `snapshot-max-count` 设置的值。更改 `snapshot-max-count` 的值不会影响现有卷。

=== 卷特定设置

*快照最大数量*:您可以为特定卷创建的最大快照数量。

您可以指定“0”或任何介于“2”和“250”之间的值。默认值为“0”。

当您创建一个卷而不更改此设置的默认值时，Longhorn 会应用 `snapshot-max-count` 设置的值。

*快照最大大小*:特定卷的快照最大聚合大小。

您可以指定 "0" 或任何大于 `Volume.Spec.Size` 乘以 2 的值。您必须将 `Volume.Spec.Size` 的值加倍，因为 Longhorn 至少需要两个快照才能正常工作。

默认值为“0”，这实际上禁用了该设置。

当您扩展卷大小时，Longhorn 会自动将该设置的值增加到 `Volume.Spec.Size` 乘以 2（如果当前值较小）。
