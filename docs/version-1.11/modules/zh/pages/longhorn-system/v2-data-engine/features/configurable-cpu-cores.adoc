= 可配置的 CPU 核心
:revdate: 2026-03-16
:page-revdate: {revdate}

{longhorn-product-name} 现在通过使用 *CPU 掩码* 支持 V2 数据引擎的可配置 CPU 核心。该掩码允许您精确定义分配给引擎的 CPU 核心，提供全局和每节点的配置选项。

== 了解 CPU 掩码

CPU 掩码是一个 *十六进制 (hex) 表示的二进制位掩码*，其中每个位对应一个特定的 CPU 核心。

* 设置为 *1* 的位表示该 CPU 核心已启用数据引擎。
* 设置为 *0* 的位表示该 CPU 核心被跳过。

=== 如何计算掩码

要确定正确的十六进制字符串，请将您的 CPU 核心视为从右到左的一系列位：

[cols="1,1,1"]
|===
| 所需 CPU 核心数量 | 二进制表示 (CPU 核心 3, 2, 1, 0) | 十六进制值

| *1 个 CPU 核心*
| `0001`
| *0x1*

| *2 个 CPU 核心*
| `0011`
| *0x3*

| *3 个 CPU 核心*
| `0111`
| *0x7*

| *4 个 CPU 核心*
| `1111`
| *0xF*
|===

*23 个 CPU 核心的示例*：要分配 23 个 CPU 核心，您需要设置 23 位为 `1`。

* *二进制*: `111 1111 1111 1111 1111 1111`
* *十六进制*: `0x7FFFFF`

[NOTE]
====
不要将核心数量与十六进制值混淆。例如，设置 `0x4` (二进制 `0100`) 仅启用 *一个* CPU 核心 (核心 #2)，而 `0xF` (二进制 `1111`) 启用 *四个* CPU 核心。
====

== 全局配置

要在整个集群中全局设置 CPU 核心，请更新 xref:longhorn-system/settings.adoc#_data_engine_cpu_mask[`data-engine-cpu-mask`] 设置。

1. 导航到 *设置 > 常规*。
2. 找到 *数据引擎 CPU 掩码*。
3. 输入您计算的十六进制字符串（例如，`0xF`）。

== 每节点 CPU 核心配置

对于特定节点的 CPU 核心分配，请使用十六进制编码字符串更新实例管理器的 `spec.dataEngineSpec.v2.cpuMask` 字段。默认情况下，此值为空，v2 数据引擎将使用 `data-engine-cpu-mask` 指定的全局设置。当设置每节点配置时，v2 数据引擎将优先考虑该特定节点的此值，而不是全局设置。

== 计算工具

您可以使用 https://www.rapidtables.com/convert/number/binary-to-hex.html[二进制转十六进制转换器] 来帮助计算您的掩码。为每个您希望分配的 CPU 核心输入一个 `1`，并将结果二进制字符串转换为十六进制。
