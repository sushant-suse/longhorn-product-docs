= Accédez à l'interface utilisateur
:revdate: 2026-03-17
:page-revdate: {revdate}
:current-version: {page-component-version}

L'exposition de {longhorn-product-name} l'interface utilisateur permet un accès externe à la console de gestion. Le choix du contrôleur Ingress (par exemple, *Traefik*, *HAProxy*, etc.) n'affecte que la manière dont l'interface utilisateur est accessible. Cela n'a *aucun impact sur le {longhorn-product-name} backend, les opérations de stockage ou l'intégrité des données*.

[NOTE]
====
À partir de novembre 2025, le projet Kubernetes a annoncé le retrait du contrôleur ingress-nginx. Pour plus de détails, https://kubernetes.io/blog/2025/11/11/ingress-nginx-retirement[consultez l'annonce officielle].
====

== Conditions préalables à l'accès et à l'authentification

Ces instructions supposent que {longhorn-product-name} est déjà installé dans le cluster.

Si vous avez installé {longhorn-product-name} en utilisant le manifeste YAML, vous devez configurer un contrôleur Ingress pour permettre le trafic externe dans le cluster. L'authentification n'est *pas activée par défaut*. Cela s'applique à la fois aux installations Helm et `kubectl`.

Pour des informations sur la création d'un contrôleur Ingress Traefik avec authentification de base, consultez xref:longhorn-system/system-access/longhorn-ingress-traefik.adoc[cette section]. Alternativement, vous pouvez également utiliser le xref:longhorn-system/system-access/longhorn-httproute.adoc[Gateway API HTTPRoute] comme une approche moderne pour exposer {longhorn-product-name} l'interface utilisateur.

Si {longhorn-product-name} a été installé en tant qu'application de catalogue Rancher, Rancher crée automatiquement un contrôleur Ingress avec contrôle d'accès (le `rancher-proxy`).

== Accès à {longhorn-product-name} l'interface utilisateur

Après l'installation de {longhorn-product-name} dans votre cluster Kubernetes, vous pouvez accéder au tableau de bord de l'interface utilisateur en suivant ces étapes.

. Récupérez les informations du service {longhorn-product-name} :
+
[,shell]
----
kubectl -n longhorn-system get svc
----
+
Pour {longhorn-product-name} v0.8.0 et versions ultérieures, la sortie ressemble à ce qui suit. Utilisez le `CLUSTER-IP` du service `longhorn-frontend` pour accéder à l'interface utilisateur :
+
[,shell]
----
NAME                TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)    AGE
longhorn-backend    ClusterIP   10.20.248.250   <none>        9500/TCP   58m
longhorn-frontend   ClusterIP   10.20.245.110   <none>        80/TCP     58m
----
+
Dans cet exemple, l'interface utilisateur est accessible à `10.20.245.110`.
+
[NOTE]
====
À partir de {longhorn-product-name} v0.8.0, le type de service de l'interface utilisateur a changé de `LoadBalancer` à `ClusterIP`.
====

. Ouvrez un navigateur et accédez à l'adresse IP du service `longhorn-frontend`.
+
