== Versionsbezeichnungen

Jede kleinere Version wird unterstützt, bis die nächste kleinere Version stabil wird.

* *Dev*: Die Software befindet sich in aktiver Entwicklung und wurde nicht gründlich getestet. Das Upgrade von früheren Versionen oder auf spätere Versionen wird nicht unterstützt. "Entwicklung"-Versionen treten typischerweise am Ende eines Sprints auf und können auch erscheinen, wenn andere Definitionen von "fertig" erfüllt sind. Angesichts der inoffiziellen Natur der Version verwenden Sie die Software nur zu Testzwecken.

* *Neueste*: Die Software hat alle Phasen der Verifizierung und des Testens bestanden. Alle verbleibenden bekannten Probleme werden als akzeptabel angesehen. Verwenden Sie die Software nur zu Testzwecken.

* *Stabil*: Die Software wird als zuverlässig und frei von kritischen Problemen angesehen. Diese Version wird für den allgemeinen Gebrauch empfohlen.

* *EOL*: Die Software hat das Ende ihrer Nutzungsdauer erreicht und es wird keine weitere Wartung auf Codeebene bereitgestellt. Sie können die Software weiterhin im Rahmen der Lizenzvereinbarung verwenden. Ein Upgrade auf eine stabile Version wird empfohlen.

== Funktionsbezeichnungen

Funktionen, die als **Experimentell** und **Technische Vorschau** gekennzeichnet sind, bieten Einblicke in bevorstehende Innovationen und bieten Möglichkeiten, neue Technologien in Ihrer Umgebung zu testen.

* *Experimentell*: Die Funktion ist unvollständig, aber ihre wesentliche Funktionalität ist betriebsbereit. Tests in einer kontrollierten Umgebung haben gezeigt, dass sie mit bestehenden stabilen Funktionen koexistieren kann. Aufgrund der fehlenden Komponenten und/oder der sich entwickelnden Funktionalität wird die Funktion nicht für den Einsatz in Produktionsumgebungen empfohlen.

* *Technische Vorschau*: Die Funktion ist fast vollständig und ihre Funktionalität wird voraussichtlich nicht erheblich verändert. Tests in einer kontrollierten Umgebung haben gezeigt, dass sie mit bestehenden stabilen Funktionen koexistieren kann. Erforschen Sie die Funktion umfassend, bevor Sie sie in Produktionsumgebungen verwenden.

Diese Funktionen haben die folgenden Einschränkungen:

* Noch in der Entwicklung und möglicherweise funktional unvollständig, instabil oder auf andere Weise für die Produktionsnutzung ungeeignet.
* Nicht unterstützt.
* Möglicherweise nur für bestimmte Hardwarearchitekturen verfügbar. Details und Funktionalität können sich ändern. Daher ist ein Upgrade auf nachfolgende Versionen möglicherweise nicht möglich und erfordert möglicherweise eine Neuinstallation.
* Kann jederzeit aus dem Produkt entfernt werden. Dies kann beispielsweise auftreten, wenn wir feststellen, dass die Funktion nicht den Kunden- oder Marktbedürfnissen entspricht oder nicht den Unternehmensstandards genügt.

[NOTE]
====
*Experimentell* und *Technische Vorschau* Funktionen sind in der Regel standardmäßig deaktiviert. Die Dokumentation bietet Informationen darüber, wie Sie solche Funktionen aktivieren und konfigurieren können.
====

== Hinweise

Hinweise sind Textblöcke, die zusätzliche oder wichtige Informationen bereitstellen. Jeder Textblock ist mit einem speziellen Label und Symbol gekennzeichnet.

* *Hinweis:* Zusätzliche Informationen, die nützlich, aber nicht entscheidend sind.

* *Wichtig*: Informationen, die besondere Aufmerksamkeit erfordern, wie Änderungen im Produktverhalten.

* *Tipp*: Hilfreiche Ratschläge und Vorschläge, wie alternative Methoden zur Durchführung einer Aufgabe.

* *Achtung*: Informationen, die vor dem Fortfahren mit bestimmten Aktionen berücksichtigt werden müssen, wie Fehler, die auftreten können.

* *Warnung*: Kritische Informationen über schädliche Folgen, wie irreversible Schäden und Datenverlust.
