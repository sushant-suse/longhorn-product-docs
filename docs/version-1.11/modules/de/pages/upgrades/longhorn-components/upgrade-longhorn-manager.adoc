= Upgrade Longhorn Manager
:revdate: 2026-03-24
:page-revdate: {revdate}
:current-version: {page-component-version}

[WARNING]
====
Es wird empfohlen, eine xref:snapshots-backups/system-backups/create-system-backup.adoc[Longhorn-System-Sicherung] zu erstellen, bevor das Upgrade durchgeführt wird. Dies stellt sicher, dass alle kritischen Ressourcen, wie Volumes und Backing-Images, gesichert werden und im Falle von Problemen wiederhergestellt werden können.
====

== Upgrade von v1.10.x

{longhorn-product-name} unterstützt nur das Upgrade auf v{current-version} von v1.10.x. Für andere Versionen führen Sie bitte zunächst ein Upgrade auf v1.10.x durch.

Für Air-Gapped-Upgrades, wenn {longhorn-product-name} als Rancher-App installiert ist, müssen Sie die Bildnamen ändern und den Teil der Registrierungs-URL entfernen.

Für Air-Gapped-Upgrades, wenn {longhorn-product-name} als Rancher-App installiert ist, müssen Sie die Bildnamen ändern und den Teil der Registrierungs-URL entfernen. Zum Beispiel wird das Bild `registry.example.com/longhorn/longhorn-manager:v{patch-version}` im Abschnitt Longhorn-Bilder zu `longhorn/longhorn-manager:v{patch-version}` geändert. Für weitere Informationen siehe die Schritte zur Installation im Air Gap xref:installation-setup/installation/airgapped-environment.adoc#_using_a_rancher_app[hier].

=== Vorbereitung auf das Upgrade

Die aktuelle Standard-StorageClass hat die folgenden Parameter:

[,yaml]
----
parameters:
  numberOfReplicas: <user specified replica count, 3 by default>
  staleReplicaTimeout: "30"
  fromBackup: ""
  baseImage: ""
----

== Aufrüsten

[IMPORTANT]
====
Sichern Sie immer Volumes, bevor Sie ein Upgrade durchführen. Wenn etwas schiefgeht, können Sie das Volume mit der Sicherung wiederherstellen.
====

=== Upgrade über die {rancher-product-name} UI

Für Kubernetes-Cluster, die von {rancher-product-name} verwaltet werden, können Sie die {longhorn-product-name} App aktualisieren, indem Sie das Repository aktualisieren, um auf eine neuere Helm-Chart-Version zu verweisen. 

Im folgenden Verfahren wird beschrieben, wie Sie die {longhorn-product-name} App auf {rancher-product-name} upgraden.

. Gehen Sie zu *Apps > Repositories*.
. Klicken Sie auf *Erstellen*.
. Wählen Sie *OCI Repository* als Ziel.
. Geben Sie einen Namen für das Repository an, zum Beispiel `suse-storage`.
. Geben Sie im Feld *OCI Repository Host URL* ein:
+
`oci://dp.apps.rancher.io/charts/suse-storage`
+
image::screenshots/install/add-repo.png[Repository hinzufügen, width=100%]

. Klicken Sie auf die Schaltfläche *Erstellen*.
. Überprüfen Sie, ob das Repository erfolgreich hinzugefügt wurde.
+
image::screenshots/install/add-repo-success.png[Repository hinzugefügt, width=100%]

. Gehen Sie zu *Apps > Charts* und suchen Sie das `suse-storage` Chart.
+
image::screenshots/install/find-chart.png[Chart finden, width=100%]

. Klicken Sie auf das Chart und dann auf *Installieren*.
+
image::screenshots/install/install-chart.png[Chart installieren, width=100%]

. Setzen Sie auf der nächsten Seite `global.imagePullSecrets` auf `application-collection` und klicken Sie dann auf *Installieren*.
+
Verweisen Sie auf die https://docs.apps.rancher.io/get-started/authentication/#kubernetes[Authentifizierungsdokumentation] für die Einrichtung der Anmeldeinformationen.
+
[,yaml]
----
global:
  cattle:
    windowsCluster:
      defaultSetting:
        systemManagedComponentsNodeSelector: kubernetes.io/os:linux
        taintToleration: cattle.io/os=linux:NoSchedule
      enabled: false
      nodeSelector:
        kubernetes.io/os: linux
      tolerations:
        - effect: NoSchedule
          key: cattle.io/os
          operator: Equal
          value: linux
  imagePullSecrets: [application-collection]
  imageRegistry: ''
----

. Dies aktualisiert `suse-storage` `{current-version}`
+
image::screenshots/install/update-repo.png[Aktualisierungs-Repository, width=100%]

. Gehen Sie nach dem Upgrade zu *Apps > Repositories*.
+
[NOTE]
.Für {rancher-product-name} Versionen vor v2.13 befolgen Sie diese Schritte:
====
[lowerroman]
. Wählen Sie das `suse-storage` Repository aus, klicken Sie auf das Drei-Punkte-Menü und wählen Sie *Konfiguration bearbeiten*.
. *Behelfslösung:* Aktualisieren Sie die URL auf `oci://dp.apps.rancher.io/charts/suse-storage:{patch-version}`, um sicherzustellen, dass die Version erkannt wird.
. Das Repository sollte erfolgreich aktualisiert werden.
+
image::screenshots/install/updated-repo.png[Aktualisierungs-Repository, width=100%]

. Gehen Sie zu *Apps > Installierte Apps*.
. Im `longhorn-system` Namespace, unter der *Upgradebar* Spalte, suchen Sie nach einem gelb hervorgehobenen `{patch-version}` Etikett.
+
image::screenshots/install/upgradable-column.png[Upgradebare Version, width=100%]

. Klicken Sie auf das gelbe `{patch-version}` Etikett, um die Upgrade-Seite zu öffnen.
+
image::screenshots/install/upgrade-page.png[Upgrade-Seite, width=100%]

. Klicken Sie auf *Weiter*.
. Stellen Sie sicher, dass `global.imagePullSecrets` auf `application-collection` eingestellt ist, und klicken Sie dann auf *Upgrade*.
+
image::screenshots/install/upgrade-progress.png[Upgrade-Fortschritt, width=100%]
====
+
. Gehen Sie zu *Apps > Charts* und wählen Sie das `suse-storage` Chart aus.
. Wählen Sie die gewünschte Upgrade-Version aus dem *Chart-Versionen* Panel aus.
+
image::screenshots/install/current-version-rancher-2-13_and_above.png[Aktualisierungs-Repository in Rancher 2.13+, width=100%]

. Nachdem das Upgrade abgeschlossen ist, überprüfen Sie, ob die `suse-storage` Chart-Version auf `{patch-version}` aktualisiert wurde.

=== Upgrade mit Helm

Das Upgrade des {longhorn-product-name} Helm-Diagramms umfasst die Aktualisierung Ihrer Bereitstellung auf eine neuere Version oder den Wechsel zwischen Longhorn-Community-Images und {longhorn-product-name} Images.

Der allgemeine Helm-Befehl zum Upgrade lautet:

[subs="+attributes",shell]
----
helm upgrade longhorn oci://dp.apps.rancher.io/charts/suse-storage \
  --namespace longhorn-system \
  --version <version> \ # Replace with the version you would like to upgrade to
  --set global.imagePullSecrets=<PULL_IMAGE_SECRET> \
  -f values.yaml
----

[NOTE]
====
Um ein Geheimnis zu erstellen, folgen Sie der https://docs.apps.rancher.io/get-started/authentication/#kubernetes[AppCo-Dokumentation].
====

Oder, wenn Sie ohne eine spezifische `values.yaml` Datei upgraden und nur die Version ändern:

[subs="+attributes",shell]
----
helm upgrade longhorn oci://dp.apps.rancher.io/charts/suse-storage \
  --namespace longhorn-system \
  --version <version> \ # Replace with the version you would like to upgrade to
  --set global.imagePullSecrets=<PULL_IMAGE_SECRET>
----

* *Upgrade-Pfad*: Verweisen Sie auf die offizielle xref:upgrades/upgrades.adoc[{longhorn-product-name} Upgrade-Dokumentation] für spezifische Version-zu-Version Upgrade-Pfade, Voraussetzungen und Schritte zur Überprüfung nach dem Upgrade. Dies ist wichtig, um ein reibungsloses Upgrade und die Datenintegrität sicherzustellen.
* *Konfigurationsänderungen*: Wenn Sie benutzerdefinierte Konfigurationen haben, stellen Sie sicher, dass Sie diese in die neue Chart-Version übernehmen und mit allen Standardänderungen zusammenführen.

=== Upgrade mit Fleet

Aktualisieren Sie den Wert von `helm.version` in der `fleet` YAML-Datei Ihres GitOps-Repositorys.

[subs="+attributes",yaml]
----
helm:
  repo: https://charts.longhorn.io
  chart: longhorn
  version: v{patch-version} # Replace with the {longhorn-product-name} version you would like to upgrade to
  releaseName: longhorn
----

=== Upgrade mit Flux

Aktualisieren Sie den Wert von `spec.chart.spec.version` in der `HelmRelease` YAML-Datei Ihres GitOps-Repositorys.

[subs="+attributes",yaml]
----
spec:
  chart:
    spec:
      chart: longhorn
      reconcileStrategy: ChartVersion
      sourceRef:
        kind: HelmRepository
        name: longhorn
      version: v{patch-version} # Replace with the {longhorn-product-name} version you would like to upgrade to
----

=== Upgrade mit Argo CD

Aktualisieren Sie den Wert von `targetRevision` in der `Application` YAML-Datei Ihres GitOps-Repositorys.

[subs="+attributes",yaml]
----
spec:
  project: default
  sources:
    - chart: longhorn
      repoURL: https://charts.longhorn.io
      targetRevision: v{patch-version} # Replace with the {longhorn-product-name} version you would like to upgrade to
----
